# TabletopForge — Customization Guide

## 1. Colours

All colours are CSS variables in `assets/css/style.css` under `:root`.

### Change the accent colour

Search for `#D4F53C` — it appears in one place as the `--accent` token:

```css
--accent       : #D4F53C;   /* ← Change this */
--accent-muted : rgba(212, 245, 60, 0.10);  /* ← Update the RGB values to match */
--accent-border: rgba(212, 245, 60, 0.25);  /* ← Update the RGB values to match */
```

Use an RGBA calculator to convert your new hex to the alpha versions.

### Switch to light mode by default

In every HTML file, change the `<html>` opening tag:

```html
<!-- Dark (current default) -->
<html lang="en" data-theme="dark">

<!-- Light default -->
<html lang="en" data-theme="light">
```

### Modify section background gradients

Open `assets/css/style.css`. The bundled page sections are clearly labelled, e.g.:

```css
/* TabletopForge — visual-overhaul.css */
.campaigns {
  background:
    radial-gradient(ellipse 60% 50% at 0% 0%, rgba(212, 245, 60, 0.028) 0%, transparent 60%),
    var(--bg-primary);
}
```

Adjust the `rgba(212, 245, 60, 0.028)` opacity (last value) to increase/decrease the glow intensity.
Set to `0` to remove it entirely.

## 2. Typography

Fonts are loaded from Google Fonts in `style.css`:

```css
@import url('https://fonts.googleapis.com/css2?family=Syne:wght@400;600;700;800&family=DM+Sans...');
```

Two variables control all typography:

```css
--font-head : 'Syne', sans-serif;   /* Headings, labels, numbers */
--font-body : 'DM Sans', sans-serif; /* Body text, buttons, inputs */
```

To change fonts: update the `@import` URL and the two variables.

## 3. Content Replacement

### Studio name
Search globally for `TabletopForge` — update in all HTML files and CSS comments.

### Logo SVG
The nav logo is an inline SVG in every `<header>`. Replace the `<svg>` paths with your own icon, or swap to an `<img>` tag:

```html
<!-- Replace the SVG with an image -->
<div class="nav__logo-mark">
  <img src="../assets/images/logo-mark.svg" alt="" width="18" height="18" />
</div>
```

### Accent colour brand name dot (footer)
The lime dot before the brand name in the footer is CSS-generated by `style.css` / `atmosphere.css`:
```css
.footer__brand-name::before { background: var(--accent); }
```
Remove the `::before` rule to hide it.

## 4. Campaign Data

All campaign numbers are hardcoded HTML. Key locations:

| Data | File | Element |
|------|------|---------|
| Funded % | `game-single.html` | `data-count="342" data-suffix="%"` |
| Backers | `game-single.html` | `data-count="2847"` |
| Days left | `game-single.html` | `data-count="14"` |
| Progress fill | Any `.progress__fill` | `data-width="100"` (percentage) |

The `CounterManager` in `main.js` animates any element with `data-count`. The value you put in is the target number.

## 5. Navigation

The nav is duplicated across pages (no server-side includes). `_nav.html` is a reference copy.

When adding a new nav link:
1. Update every HTML file's `<header>` block
2. Update every `<nav class="nav__mobile">` block
3. Update `_nav.html` as the canonical reference

## 6. Dashboard Panels

`dashboard-user.html` uses an in-page panel system.
The admin area uses dedicated section pages such as `dashboard-admin-analytics.html`,
`dashboard-admin-orders.html`, and `dashboard-admin-settings.html`.

To add a new user dashboard panel:
```html
<!-- 1. Add nav item in sidebar -->
<a href="#newpanel" class="dash-nav__item" data-panel="newpanel">
  <i class="fa-solid fa-star"></i> New Panel
</a>

<!-- 2. Add panel content -->
<div class="dash-page" id="dash-panel-newpanel">
  <!-- Your content here -->
</div>
```

`dashboard.js` handles the switching automatically for `data-panel` items.
For the admin area, duplicate the closest existing `dashboard-admin-*.html` file and
update the sidebar link target plus the single visible `admin-panel-*` block.

## 7. Animated Counters

Any element:
```html
<span data-count="48" data-suffix="K+">—</span>
<span data-count="2.4" data-prefix="$" data-suffix="M">—</span>
```

The `—` is the placeholder shown before animation. Triggered on scroll by `CounterManager` (homepage/about) or `animateCounters()` (dashboards).

---

## 8. Adding a Blog Post

1. Duplicate `blog-single.html`
2. Update the `<title>`, meta description, `<h1>`, author, date, and article body
3. Add to `blog.html` grid as a new `.post-card`
4. Add to `sitemap.xml`
